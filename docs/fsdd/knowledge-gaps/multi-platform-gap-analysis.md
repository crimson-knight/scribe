## Part 1: Current Scribe Compilation Status

### Does It Compile?

| Target | Status | Blocking Issue |
|--------|--------|---------------|
| **macOS** | YES (compiles and runs) | Amber schema examples print noise to stdout on startup |
| **iOS** | NO | No Xcode project, cross-compile deps not built |

### Root Cause

`src/scribe.cr` was generated by the Amber V2 CLI for a web app:
```crystal
require "../config/*"          # config/ exists but has web-only setup
require "./controllers/**"      # Only application_controller.cr exists
require "./models/**"           # Empty
require "./schemas/**"          # Missing directory entirely
require "./jobs/**"             # Missing directory entirely
require "./mailers/**"          # Missing directory entirely
require "./channels/**"         # Missing directory entirely

Amber::Server.start             # Starts HTTP server — wrong for native app
```

### What Must Change in src/scribe.cr

```crystal
require "amber"
require "asset_pipeline/ui"

# Amber patterns without HTTP server
Amber.settings.name = "Scribe"

# Platform-specific requires
{% if flag?(:macos) %}
  require "./platform/macos/**"
{% elsif flag?(:ios) %}
  require "./platform/ios/**"
{% elsif flag?(:android) %}
  require "./platform/android/**"
{% end %}

require "./models/**"
require "./process_managers/**"
require "./controllers/**"
require "./ui/**"

# Native event loop (NOT Amber::Server.start)
{% if flag?(:macos) %}
  # CFRunLoop / NSApplication event loop
{% elsif flag?(:ios) %}
  # UIApplication main loop (called from Swift/ObjC host)
{% elsif flag?(:android) %}
  # Android Activity lifecycle (called from JNI host)
{% end %}
```

## Part 2: Amber V2 CLI Gaps for Multi-Platform

### What the CLI Generates Today

The `amber new <app> -d sqlite` command generates a **web-only** scaffold:
- Single target in shard.yml
- `Amber::Server.start` in main entry point
- HTTP middleware pipeline in routes.cr
- HTML views (ECR or Slang templates)
- No compile-time flags or platform awareness

### What It Would Need for Multi-Platform

#### New CLI Flag: `--type`
```bash
amber new my_app -d sqlite --type native --targets macos,ios,android
```

#### Modified `.amber.yml`
```yaml
app: my_app
type: native              # NEW: web | native
database: sqlite
targets:                  # NEW: platform target config
  macos:
    renderer: appkit
    build_flags: [-Dmacos]
    link_frameworks: [AppKit, Foundation, AVFoundation, CoreAudio]
  ios:
    renderer: uikit
    build_flags: [-Dios]
    cross_compile: aarch64-apple-ios17.0
  android:
    renderer: android
    build_flags: [-Dandroid]
    cross_compile: aarch64-linux-android31
```

#### Modified Main Entry Point
When `--type native`, generate:
```crystal
require "amber"
require "asset_pipeline/ui"
require "../config/application"
require "./models/**"
require "./process_managers/**"
require "./controllers/**"
require "./ui/**"

{% if flag?(:macos) %}
  require "./platform/macos/app"
{% elsif flag?(:ios) %}
  require "./platform/ios/app"
{% elsif flag?(:android) %}
  require "./platform/android/app"
{% end %}
```

#### Modified config/application.cr
When `--type native`:
```crystal
require "amber"

Amber.settings.name = "my_app"
# No Amber::Server.configure — native apps don't need HTTP
```

#### Additional Generated Directories
```
src/
├── platform/
│   ├── macos/app.cr         — NSApplication event loop
│   ├── ios/app.cr           — UIApplication integration
│   └── android/app.cr       — Activity/JNI integration
├── process_managers/        — FSDD business logic
├── ui/
│   └── main_view.cr         — Asset Pipeline UI root view
└── events/                  — Internal event bus
```

#### Build Script (Makefile)
Generate a Makefile or `scripts/build.sh` with per-platform commands:
```makefile
.PHONY: macos ios android

macos: ext
	crystal-alpha build src/scribe.cr -o bin/scribe -Dmacos \
	  --link-flags="lib/asset_pipeline/src/ui/native/objc_bridge.o ..."

ios: ext
	crystal-alpha build src/scribe.cr --cross-compile \
	  --target aarch64-apple-ios17.0 --shared -Dios ...

android: ext
	crystal-alpha build src/scribe.cr --cross-compile \
	  --target aarch64-linux-android31 --shared -Dandroid ...

ext:
	clang -c lib/asset_pipeline/src/ui/native/objc_bridge.m \
	  -o lib/asset_pipeline/src/ui/native/objc_bridge.o -fno-objc-arc
	cd lib/crystal-audio && make ext
```

## Part 3: Library Documentation Gaps (shards-alpha ai-docs)

### How Skills Propagate

shards-alpha's `AIDocsInstaller` automatically copies `.claude/` content from dependencies:
- Skills: `<shard>--<name>/SKILL.md`
- Agents: `<shard>--<name>.md`
- Commands: `<shard>:<name>.md`
- CLAUDE.md: wrapped as `<shard>--docs/SKILL.md`

Tracked in `.claude/.ai-docs-info.yml` with dual checksums for safe updates.
